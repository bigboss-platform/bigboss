## 1. Language and Framework Rules

- Python **3.12** minimum. No older syntax.
- FastAPI is the only accepted web framework for BigBoss API services.
- All I/O operations must be **async** (`async def`, `await`). No synchronous database calls.
- **Pydantic v2** is used for all data validation and serialization. Never use v1 syntax.
- **SQLAlchemy 2.x** with async sessions. Never use the legacy 1.x query API.
- Type annotations are **required on every function signature** — parameters and return types.
- `pyproject.toml` is the single source of project metadata and dependencies. No `requirements.txt`.

## 2. Naming Conventions

### Files and Folders

| Item | Convention | Example |
|---|---|---|
| Module folder | `snake_case` | `tenant_admins/`, `orders/` |
| Python file | `snake_case.py` | `order_service.py` |
| Migration file | auto-generated by Alembic | `20260415_add_order_status.py` |

### Python Code

| Item | Convention | Example |
|---|---|---|
| Variable | `snake_case` | `tenant_id`, `order_total` |
| Function | `snake_case` | `get_order_by_id`, `calculate_total` |
| Class | `PascalCase` | `OrderService`, `TenantRepository` |
| Constant | `SCREAMING_SNAKE_CASE` | `MAX_ORDER_ITEMS`, `DEFAULT_PAGE_SIZE` |
| Enum | `PascalCase` class, `SCREAMING_SNAKE_CASE` members | `class OrderStatus(str, Enum): PENDING = "pending"` |
| Pydantic schema | `PascalCase` with suffix | `OrderCreateSchema`, `OrderReadSchema` |
| SQLAlchemy model | `PascalCase` no suffix | `Order`, `Tenant`, `EndUser` |
| FastAPI router | `snake_case` variable | `router = APIRouter(...)` |
| Exception class | `PascalCase` with `Exception` suffix | `OrderNotFoundException` |

### Names Must Be Descriptive

- No abbreviations: `usr` → `user`, `ord` → `order`, `qty` → `quantity`
- No single-letter variables except loop counters (`i`, `j`) in tight loops
- No generic names: `data`, `result`, `obj`, `item` alone — qualify them: `order_data`, `tenant_result`
- Function names start with a verb: `get_`, `create_`, `update_`, `delete_`, `calculate_`, `validate_`, `send_`, `resolve_`

## 3. Module Structure Rules

Every module in `app/modules/` must contain exactly these files (create them all when bootstrapping a new module, even if some are initially minimal):

```
module_name/
├── public.py          ← required — the only import surface for other modules
├── router.py          ← required — FastAPI APIRouter
├── service.py         ← required — business logic
├── repository.py      ← required — database access
├── models.py          ← required — SQLAlchemy models
├── schemas.py         ← required — Pydantic input/output schemas
├── exceptions.py      ← required — module-specific exceptions
└── dependencies.py    ← required — FastAPI dependency functions for this module
```

### Inter-module imports

```python
# CORRECT — importing through public.py
from app.modules.tenants.public import TenantService, TenantReadSchema

# WRONG — importing internal files of another module
from app.modules.tenants.service import TenantService        # forbidden
from app.modules.tenants.models import Tenant                 # forbidden
from app.modules.tenants.repository import TenantRepository   # forbidden
```

If module A needs data from module B, it calls `B`'s service — it never queries `B`'s tables directly.

## 4. API Design Rules

### URL Structure

```
/api/v1/{resource-plural}/{id}/{sub-resource-plural}/{sub-id}
```

Examples:
```
GET    /api/v1/tenants/{tenant_id}/orders
GET    /api/v1/tenants/{tenant_id}/orders/{order_id}
POST   /api/v1/tenants/{tenant_id}/orders
PUT    /api/v1/tenants/{tenant_id}/orders/{order_id}
DELETE /api/v1/tenants/{tenant_id}/orders/{order_id}
```

Rules:
- All resource names are **plural** and **kebab-case**: `order-items`, `tenant-admins`
- IDs are always UUID strings — never auto-increment integers in public APIs
- API versioning is in the URL prefix (`/api/v1/`)
- Never use verbs in URLs — use HTTP methods instead: `POST /orders` not `GET /create-order`

### Relationship Modules — Domain Noun Rule

When two resources have a relationship (e.g. a user subscribes to a newsletter, a member joins a gym), **never add endpoints to the existing resource modules**. Instead, identify the domain noun that the relationship produces and give it its own module with standard CRUD endpoints.

| Domain | Wrong — pollutes existing module | Right — new domain noun module |
|--------|----------------------------------|-------------------------------|
| Gym | `POST /users/:id/join-gym` | `POST /memberships` |
| Uber-like | `POST /drivers/:id/accept-trip` | `POST /trips`, `POST /assignments` |
| Medical | `POST /doctors/:id/see-patient` | `POST /appointments` |
| Newspaper | `POST /users/:id/subscribe` | `POST /subscriptions` |
| Any | `POST /users/:id/pay` | `POST /payments` |

The relationship's foreign keys (`user_id`, `plan_id`, etc.) are fields on the new resource — not URL params on the parent.

```
# WRONG — bolts subscription logic onto the User module
POST   /users/{user_id}/subscribe
DELETE /users/{user_id}/unsubscribe

# CORRECT — Subscription is a first-class resource
POST   /api/v1/subscriptions                         ← create (subscribe)
GET    /api/v1/subscriptions/{subscription_id}       ← read (check status)
PATCH  /api/v1/subscriptions/{subscription_id}       ← update (change plan)
DELETE /api/v1/subscriptions/{subscription_id}       ← delete (unsubscribe)
GET    /api/v1/users/{user_id}/subscriptions         ← list user's subscriptions (read collection)
```

**Naming rule:** the module name is a **domain noun**, never a verb phrase. `Subscription` not `UserCanSubscribeToNewspaper`. `Membership` not `UserCanJoinGym`.

### Action Sub-Routes — State Transitions

Some state transitions carry business meaning that goes beyond a simple field update. Give these a named sub-route rather than forcing them into `PATCH`:

```
POST /api/v1/subscriptions/{id}/pause
POST /api/v1/subscriptions/{id}/resume
POST /api/v1/trips/{id}/cancel
POST /api/v1/appointments/{id}/confirm
POST /api/v1/orders/{id}/cancel
```

**Rule:** use an action sub-route when the transition has side effects (emails, notifications, billing events) or when the state machine makes the transition name semantically meaningful beyond "update a field".

### HTTP Methods

| Method | Use |
|---|---|
| `GET` | Retrieve — never modifies state |
| `POST` | Create a new resource |
| `PUT` | Replace a resource completely |
| `PATCH` | Partial update |
| `DELETE` | Remove a resource |

### Response Shape

All responses follow a consistent envelope:

```python
# Success — list
{
    "data": [...],
    "meta": {
        "total": 100,
        "page": 1,
        "page_size": 20
    }
}

# Success — single item
{
    "data": { ... }
}

# Error — RFC 7807 Problem Detail
{
    "type": "https://bigboss.io/errors/order-not-found",
    "title": "Order Not Found",
    "status": 404,
    "detail": "Order with id 'abc123' does not exist for this tenant.",
    "instance": "/api/v1/orders/abc123"
}
```

### Status Codes

| Situation | Code |
|---|---|
| Successful read | 200 |
| Successful creation | 201 |
| Successful delete (no content) | 204 |
| Validation error (bad input) | 422 |
| Not found | 404 |
| Unauthorized (not authenticated) | 401 |
| Forbidden (authenticated but not allowed) | 403 |
| Conflict (duplicate, constraint violation) | 409 |
| Internal server error | 500 |

## 5. Database and ORM Rules

### Model Conventions

Every SQLAlchemy model must inherit from `Base` (from `app.shared.base_model`) and include:

```python
class Order(Base):
    __tablename__ = "orders"

    id: Mapped[str] = mapped_column(String, primary_key=True, default=lambda: str(uuid4()))
    created_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), server_default=func.now())
    updated_at: Mapped[datetime] = mapped_column(DateTime(timezone=True), onupdate=func.now())
    is_deleted: Mapped[bool] = mapped_column(Boolean, default=False)
```

Rules:
- All primary keys are UUID strings
- All tables have `created_at`, `updated_at`, `is_deleted` columns
- Soft delete only — never hard-delete records. Use `is_deleted = True`.
- All timestamps are timezone-aware (`DateTime(timezone=True)`)
- Column names are `snake_case`
- Foreign keys are named `{referenced_table_singular}_id`: `tenant_id`, `end_user_id`

### Queries

```python
# CORRECT — async SQLAlchemy 2.x style
result = await session.execute(
    select(Order)
    .where(Order.tenant_id == tenant_id)
    .where(Order.is_deleted == False)
    .order_by(Order.created_at.desc())
)
orders = result.scalars().all()

# WRONG — never use legacy query API
session.query(Order).filter(Order.tenant_id == tenant_id).all()  # forbidden
```

### N+1 Query Prevention

Always use `selectinload` or `joinedload` for relationships — never access lazy-loaded relationships outside of the session:

```python
result = await session.execute(
    select(Order)
    .options(selectinload(Order.items))
    .where(Order.id == order_id)
)
```

### Schema Switching (Multi-Tenancy)

The database session automatically sets `search_path` per request based on `tenant_id` from the JWT. This is handled in `app/core/middleware.py`. No module-level code needs to handle this manually.

## 6. Schema and Validation Rules

### Schema Hierarchy Per Resource

Every resource has these Pydantic schemas:

```python
class OrderBaseSchema(BaseModel):
    # fields shared across all operations
    notes: str = ""

class OrderCreateSchema(OrderBaseSchema):
    # fields required at creation only
    end_user_id: str
    items: list[OrderItemCreateSchema]

class OrderUpdateSchema(OrderBaseSchema):
    # fields that can be updated (all optional)
    notes: str | None = None

class OrderReadSchema(OrderBaseSchema):
    # all fields returned to the client
    id: str
    status: OrderStatus
    total: Decimal
    created_at: datetime

    model_config = ConfigDict(from_attributes=True)
```

Rules:
- Use `model_config = ConfigDict(from_attributes=True)` only on Read schemas
- Never expose internal fields (like `is_deleted`, `tenant_id`) in Read schemas returned to end users
- All schema fields must have a default value or be explicitly required — no ambiguous optionals
- Use `Decimal` for money, never `float`
- Use `datetime` with timezone, never naive datetimes

### Input Validation

Validate at the schema level using Pydantic validators:

```python
from pydantic import field_validator

class OrderCreateSchema(BaseModel):
    quantity: int

    @field_validator("quantity")
    @classmethod
    def quantity_must_be_positive(cls, value: int) -> int:
        if value <= 0:
            raise ValueError("quantity must be greater than zero")
        return value
```

## 7. Service Layer Rules

- Service classes receive their dependencies via `__init__` (constructor injection)
- Services contain **only business logic** — no direct SQL, no HTTP context (no `Request` object)
- Services call repositories for data access
- Services call other module services through `public.py`
- Services raise module-specific exceptions — never FastAPI `HTTPException`
- Exception translation from service exceptions to HTTP responses happens in the router layer

```python
# CORRECT — service raises domain exception
class OrderService:
    async def get_by_id(self, order_id: str) -> OrderReadSchema:
        order = await self._repository.find_by_id(order_id)
        if order is None:
            raise OrderNotFoundException(order_id)
        return OrderReadSchema.model_validate(order)

# CORRECT — router translates to HTTP
@router.get("/{order_id}", response_model=OrderReadSchema)
async def get_order(order_id: str, service: OrderService = Depends(get_order_service)):
    return await service.get_by_id(order_id)

# exception handler in core/exceptions.py maps OrderNotFoundException → 404
```

## 8. Repository Layer Rules

- Repository classes receive an async SQLAlchemy `AsyncSession` via `__init__`
- Repositories contain **only data access logic** — no business rules
- All public methods return either an ORM model instance, a list of instances, or `None`
- Repositories never return raw dicts or tuples — always typed ORM objects
- Method names follow CRUD conventions: `find_by_id`, `find_all`, `find_by_tenant`, `create`, `update`, `soft_delete`

```python
class OrderRepository(BaseRepository[Order]):
    async def find_by_tenant(
        self,
        tenant_id: str,
        page: int = 1,
        page_size: int = 20
    ) -> tuple[list[Order], int]:
        query = (
            select(Order)
            .where(Order.tenant_id == tenant_id)
            .where(Order.is_deleted == False)
        )
        total_result = await self._session.execute(select(func.count()).select_from(query.subquery()))
        total = total_result.scalar_one()

        items_result = await self._session.execute(
            query.offset((page - 1) * page_size).limit(page_size)
        )
        return items_result.scalars().all(), total
```

## 9. Error Handling

### Exception Hierarchy

```python
# app/shared/exceptions.py
class BigBossException(Exception):
    status_code: int = 500
    error_type: str = "internal-error"
    title: str = "Internal Server Error"

class NotFoundException(BigBossException):
    status_code = 404
    error_type = "not-found"

class ConflictException(BigBossException):
    status_code = 409
    error_type = "conflict"

class ForbiddenException(BigBossException):
    status_code = 403
    error_type = "forbidden"
```

```python
# orders/exceptions.py
from app.shared.exceptions import NotFoundException, ConflictException

class OrderNotFoundException(NotFoundException):
    def __init__(self, order_id: str):
        self.detail = f"Order with id '{order_id}' does not exist."
        super().__init__(self.detail)

class OrderAlreadyCancelledException(ConflictException):
    def __init__(self, order_id: str):
        self.detail = f"Order '{order_id}' has already been cancelled and cannot be modified."
        super().__init__(self.detail)
```

A global exception handler in `app/core/exceptions.py` catches all `BigBossException` subclasses and returns the RFC 7807 response shape.

## 10. Authentication and Authorization

### Dependencies

```python
# Usage in a router
from app.core.dependencies import require_end_user, require_tenant_admin

@router.get("/profile")
async def get_profile(current_user: EndUserContext = Depends(require_end_user)):
    ...

@router.get("/dashboard")
async def get_dashboard(current_admin: TenantAdminContext = Depends(require_tenant_admin)):
    ...
```

### Context Objects

```python
@dataclass
class EndUserContext:
    end_user_id: str
    tenant_id: str
    role: str = "end_user"

@dataclass
class TenantAdminContext:
    tenant_admin_id: str
    tenant_id: str
    role: str = "tenant_admin"
```

### Authorization Rules

- Tenant isolation is enforced by the middleware (schema switching) — a tenant admin can only see their own tenant's data
- Endpoint-level permissions are enforced via dependency injection
- Never trust `tenant_id` from the request body — always take it from the JWT token context

## 11. Testing Rules

- Every module must have tests in `tests/unit/modules/{module_name}/` and `tests/integration/modules/{module_name}/`
- Unit tests test the service layer with a mocked repository
- Integration tests test the full stack against a real test database
- Test functions follow: `test_{what}_{scenario}_{expected_result}`
- Every new feature or bug fix includes a test that covers it
- Minimum coverage: **80%** per module

```python
# Example test naming
def test_get_order_by_id_when_order_exists_returns_order_schema():
    ...

def test_get_order_by_id_when_order_not_found_raises_not_found_exception():
    ...

    ...
```

## 12. Forbidden Patterns

These patterns are rejected in code review — no exceptions:

| Pattern | Why Forbidden | Correct Alternative |
|---|---|---|
| `from module.service import X` across modules | Breaks microservice boundary | Import from `module.public` |
| `session.query(Model)...` | SQLAlchemy 1.x legacy API | `select(Model)` with `session.execute` |
| `SELECT *` via `select(Model)` without filter on `is_deleted` | Returns soft-deleted records | Always add `.where(Model.is_deleted == False)` |
| `float` for monetary values | Floating point precision errors | Use `Decimal` |
| `raise HTTPException(...)` in a service | Leaks HTTP context into business logic | Raise a domain exception; router handles HTTP translation |
| Synchronous database calls (`session.execute` without `await`) | Blocks the event loop | Always `await` all async operations |
| String SQL (`text("SELECT ...")`) without parameterization | SQL injection risk | Use ORM query builder or parameterized `text()` with `bindparams` |
| Bare `except Exception:` without re-raise or logging | Swallows errors silently | Catch specific exception types or log before re-raising |
| Cross-module direct table joins | Tightly couples modules | Query through service layer |
| Magic strings inline in business logic | Untraceable, unmaintainable | Define as constants in `shared/enums.py` or module `enums.py` |
| Hard-coded configuration in source code | Not environment-portable | All config via `app/core/config.py` from environment variables |
